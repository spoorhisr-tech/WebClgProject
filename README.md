# DriveSync | Premium Vehicle Service Booking

A state-of-the-art vehicle maintenance booking platform built with **React**, **Vite**, and **Supabase**.

## 🚀 Key Features
- **Modern UI**: Sleek dark-theme interface with glassmorphism and interactive components.
- **Secure Authentication**: Built-in sign-up/login system using Supabase Auth.
- **Real-time Bookings**: Instant database synchronization and live status tracking.
- **Private Accounts**: User-based data isolation using PostgreSQL Row Level Security (RLS).
- **Responsive Design**: Fully optimized for Desktop, Tablet, and Mobile.

## 🛠️ Technology Stack
- **Frontend**: React.js / Vite
- **Backend/DB**: Supabase (PostgreSQL + Auth + Real-time)
- **Styling**: Vanilla CSS (Premium Modern Design)
- **Icons**: Lucide React

## 🏁 Getting Started

### 1. Prerequisite Documentation
Ensure you have a Supabase project set up.

### 2. Environment Setup
- Copy `.env.example` to a new file named `.env`.
- Fill in your `VITE_SUPABASE_URL` and `VITE_SUPABASE_ANON_KEY`.

### 3. Database Schema
Run the following SQL in your Supabase SQL Editor:
```sql
CREATE TABLE bookings (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  full_name TEXT NOT NULL,
  email TEXT NOT NULL,
  phone TEXT NOT NULL,
  vehicle_model TEXT NOT NULL,
  service_type TEXT NOT NULL,
  preferred_date DATE NOT NULL,
  status TEXT DEFAULT 'Scheduled',
  user_id UUID REFERENCES auth.users(id)
);

ALTER TABLE bookings ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users can insert their own bookings" ON public.bookings FOR INSERT WITH CHECK (auth.uid() = user_id);
CREATE POLICY "Users can view their own bookings" ON public.bookings FOR SELECT USING (auth.uid() = user_id);
```

### 4. Install and Run
```bash
npm install
npm run dev
```
